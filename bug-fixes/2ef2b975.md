# 로그인 페이지에서 비밀번호 입력 시 한글 입력 불가

## 로그인 페이지 비밀번호 한글 입력 버그 수정

### 1단계: 문제 원인 파악
- 비밀번호 입력 필드에 inputType='password' 설정 시 일부 브라우저에서 IME(Input Method Editor) 처리 미지원
- HTML5 password input과 한글 입력기 간 호환성 문제

### 2단계: 프론트엔드 수정
javascript
// password 필드에 IME 이벤트 핸들러 추가
const passwordInput = document.getElementById('password');

passwordInput.addEventListener('compositionstart', (e) => {
  e.target.setAttribute('data-ime-composing', 'true');
});

passwordInput.addEventListener('compositionend', (e) => {
  e.target.removeAttribute('data-ime-composing');
  e.target.dispatchEvent(new Event('input', { bubbles: true }));
});


### 3단계: HTML input 속성 확인
- autocomplete='current-password' 추가
- spellcheck='false' 설정 (브라우저 맞춤법 검사 비활성화)
- 입력 제한 regex 제거 (비밀번호는 모든 문자 허용)

### 4단계: CSS 설정
css
input[type='password'] {
  font-family: monospace;
  letter-spacing: 0.1em;
}


### 5단계: 백엔드 검증
- 서버에서 UTF-8 인코딩으로 한글 비밀번호 수용 확인
- 데이터베이스 collation UTF8MB4 또는 UTF-8 설정 확인
