## 2. 원인 분석

### 2.1 추정 원인 (Root Cause Analysis)

모바일 환경에서 첨부파일 업로드 후 폼 제출 시 무한 로딩이 발생하는 주요 원인은 다음과 같이 분류됩니다:

#### 원인 A: 업로드 상태(State) 미완료 상태에서 폼 제출 시도
```
[파일 선택] → [업로드 시작] → [업로드 완료 콜백 미실행] → [폼 제출] → [무한 로딩]
```
- 파일 업로드의 `Promise`가 resolve되지 않거나, 업로드 완료 후 `isUploading` 상태가 `false`로 전환되지 않는 경우
- 모바일에서 네트워크 응답 타임아웃 처리가 누락되어 Promise가 pending 상태로 남아있을 가능성

#### 원인 B: Race Condition (경쟁 상태)
```
[업로드 완료 이벤트] ←→ [폼 submit 이벤트] 동시 발생
```
- 업로드 완료 콜백과 폼 제출 핸들러 사이의 비동기 타이밍 이슈
- 모바일에서 터치 이벤트가 데스크탑보다 빠르게 연속 발생할 수 있음

#### 원인 C: 에러 핸들링 부재
- 모바일 환경 특유의 네트워크 불안정으로 업로드 요청 실패 시 에러 핸들링이 없어 loading 상태가 해제되지 않음
- `catch` 블록에서 `setLoading(false)` 누락
