# 로그인 페이지에서 비밀번호 입력 시 특수문자 미지원

## 비밀번호 특수문자 미지원 버그 수정

### 1단계: 입력 검증 로직 확인
- 비밀번호 입력 필드의 정규식 패턴 확인
- 현재 패턴: `/^[a-zA-Z0-9]+$/` (특수문자 제외)
- 수정 필요

### 2단계: 정규식 패턴 업데이트
- 기존 코드: `const passwordRegex = /^[a-zA-Z0-9]+$/;`
- 수정된 코드: `const passwordRegex = /^[a-zA-Z0-9!@#$%^&*()_+\-=\[\]{};:'",.<>?/\\|`~]+$/;`
- 또는 부정 접근: `const passwordRegex = /^[^\s]+$/;` (공백 제외)

### 3단계: 서버 측 검증 규칙 동기화
- 백엔드 비밀번호 검증 로직도 동일하게 업데이트
- 특수문자 인코딩/디코딩 처리 확인

### 4단계: HTML 입력 속성 확인
- `<input type="password" pattern="..." />` 제거 또는 수정
- maxlength 제한값 확인 및 조정

### 5단계: 보안 고려사항
- 강도 높은 비밀번호 검증으로 업그레이드
- XSS 및 인젝션 공격 방지 로직 추가
