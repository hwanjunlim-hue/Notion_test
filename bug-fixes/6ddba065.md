# 로그인 후 대시보드 리다이렉트 실패

## 로그인 후 대시보드 리다이렉트 실패 수정 방안

### 1단계: OAuth 콜백 핸들러 검증
- `useEffect` 또는 로그인 완료 콜백에서 토큰 저장 확인
- localStorage/sessionStorage에 인증 토큰이 정상 저장되는지 확인

### 2단계: 리다이렉트 로직 수정
javascript
// 문제 코드
if (response.ok) {
  window.location.href = '/dashboard';
}

// 수정 코드
if (response.ok) {
  localStorage.setItem('authToken', response.token);
  setTimeout(() => {
    window.location.href = '/dashboard';
  }, 100);
}


### 3단계: 대시보드 접근 권한 확인
- 보호된 라우트(PrivateRoute) 설정 검증
- 토큰 만료 시간 체크
- CORS 정책 및 API 응답 확인

### 4단계: 네트워크 요청 확인
- 브라우저 개발자 도구 Network 탭에서 리다이렉트 요청 확인
- 301/302 상태 코드 확인
- 콘솔 에러 메시지 확인

### 5단계: 세션 유지 확인
- httpOnly 쿠키 설정 검증
- SameSite 속성 확인 (CORS 환경에서 필요)
- 로그아웃 후 재로그인 테스트
