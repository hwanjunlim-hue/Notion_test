# 검색 결과 페이지 로딩 속도 저하 (5초 이상)

## 검색 결과 페이지 로딩 속도 저하 수정 방안

### 1단계: 데이터베이스 쿼리 최적화
- 검색 쿼리에 적절한 인덱스 생성 (특히 한글 검색용 전문 검색 인덱스)
- LIMIT/OFFSET 대신 커서 기반 페이지네이션 적용
- N+1 쿼리 문제 해결을 위해 JOIN 쿼리로 통합

### 2단계: API 응답 속도 개선
- 필요한 필드만 SELECT하여 데이터 크기 최소화
- 서버 사이드 캐싱 구현 (Redis 활용)
- 한글 형태소 분석기 최적화 및 캐싱

### 3단계: 프론트엔드 렌더링 최적화
- 가상 스크롤(Virtual Scrolling) 구현으로 DOM 노드 최소화
- 검색 입력값 디바운싱 적용 (300-500ms)
- 결과 이미지 lazy loading 처리

### 4단계: 네트워크 최적화
- Gzip 압축 활성화
- 번들 크기 최적화 및 코드 스플리팅
- CDN을 통한 정적 자산 배포

### 5단계: 모니터링 추가
- 로드 시간 메트릭 측정 (API 응답시간, 렌더링 시간 분리)
- 한글 검색 vs 영어 검색 성능 비교 로깅
