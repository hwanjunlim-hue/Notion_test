# 대시보드 차트 무한 로딩 버그 수정

## 문제 분석
차트 컴포넌트의 데이터 페칭 로직에서 의존성 배열이 누락되어 무한 렌더링이 발생하거나, 에러 처리 미흡으로 인한 상태 업데이트 실패로 추정됩니다.

## 수정 방안

### 1단계: 의존성 배열 확인 및 수정

```javascript
// ❌ 수정 전: useEffect 의존성 배열 누락
useEffect(() => {
  fetchChartData();
}, []); // 의존성 배열이 비어있거나 누락됨

// ✅ 수정 후: 올바른 의존성 배열
useEffect(() => {
  fetchChartData();
}, [chartId]); // 필요한 의존성 명시
```

### 2단계: 에러 처리 및 타임아웃 추가

```javascript
useEffect(() => {
  let isMounted = true;
  let timeout;

  const fetchData = async () => {
    try {
      setLoading(true);
      setError(null);

      // 타임아웃 설정 (10초)
      timeout = setTimeout(() => {
        if (isMounted) {
          setError('차트 데이터 로딩 시간 초과');
          setLoading(false);
        }
      }, 10000);

      const response = await fetch(`/api/chart/${chartId}`);
      
      if (!response.ok) {
        throw new Error(`API 오류: ${response.status}`);
      }

      const data = await response.json();
      
      if (isMounted) {
        setChartData(data);
        setLoading(false);
      }
    } catch (err) {
      if (isMounted) {
        setError(err.message);
        setLoading(false);
      }
    } finally {
      clearTimeout(timeout);
    }
  };

  fetchData();

  return () => {
    isMounted = false;
    clearTimeout(timeout);
  };
}, [chartId]);
```

### 3단계: 로딩 상태 UI 개선

```javascript
if (loading) {
  return (

if (error) {
  return (
    <div className="chart-container error">
      <p className="error-message">⚠️ {error}</p>
      <button onClick={() => fetchData()}>다시 시도</button>
    </div>
  );

return <Chart data={chartData} />;
```

### 4단계: API 응답 검증

```javascript
// API 응답 데이터 검증
if (!data || !Array.isArray(data.values)) {
  throw new Error('유효하지 않은 차트 데이터 형식');
}

if (data.values.length === 0) {
  setChartData({ ...data, values: [] });
  setError('차트 데이터가 없습니다');
  setLoading(false);
  return;
}
